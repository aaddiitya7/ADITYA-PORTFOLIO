The Challenge

After launching the A.S. Enterpriis website, GA4 was configured and deployed — property created, Measurement ID embedded, site live on Vercel. But after 7 days, the GA4 dashboard showed absolutely zero data.

No users. No sessions. No page views. The "Data collection isn't active" warning persisted even when visiting the site from multiple devices and networks.

📸 GA4 Dashboard — Zero Data Warning

The Investigation

1
Surface-Level Config Check

Verified the Measurement ID was present in code, matched the GA4 dashboard, the website URL was correct, deployment was live, and Enhanced Measurement was on. Everything looked perfect on paper — the classic trap.

2
Network-Level Debugging — The Smoking Gun

Opened browser DevTools and inspected network requests. The gtag/js script returned a 404 Not Found — Google's servers didn't recognise the Measurement ID.

Request
gtag/js?id=G-4JGJ...
Expected
200 OK (JS)
Actual
404 Not Found
3
Root Cause Analysis

The GA4 property had been migrated between Google accounts. The Data Stream's Measurement ID became orphaned — visible in the admin UI but not recognised by Google's tag-serving infrastructure. A rare post-migration edge case.

Key Takeaways

🔍
"Configured" ≠ "Working"

All visible config was correct. The failure was only detectable at the network level — a gap most marketers never check.

🌐
Always Verify at Network Level

Checking whether gtag.js?id=YOUR_ID returns 200 vs 404 is the single fastest diagnostic step.

⚠️
Account Migrations Break Streams

Moving a GA4 property between accounts can silently orphan data streams. Always re-verify after migration.

📊
Realtime Reports = Instant Feedback

GA4 Realtime is the fastest verification tool — you'll see data within seconds if tracking is live.
